# Демонстрация Git на этом проекте

Git хранит не копии папок, а цепочку зафиксированных изменений — коммитов. У этого проекта есть три состояния:

- `v1.0.0` — простая статичная заглушка;
- `v1.1.0` — добавлена подпись с номером версии;
- `v2.0.0` — добавлены JavaScript и переключатель темы;
- `v2.1.0` — добавлено сохранение выбранной темы.

## Основные команды

```cmd
git status
git log --oneline --decorate
git diff v1.0.0 v2.0.0
```

`status` показывает незаписанные изменения, `log` — историю, а `diff` — разницу между версиями.

## Посмотреть старую версию

```cmd
git switch --detach v1.0.0
```

После команды откройте `index.html`: файлы станут такими, какими были в первой версии. Чтобы вернуться к актуальной разработке:

```cmd
git switch main
```

## Создать свою экспериментальную ветку

```cmd
git switch -c experiment
```

Измените файлы, затем сохраните результат в истории:

```cmd
git add index.html styles.css
git commit -m "Describe the experiment"
```

Ветка позволяет экспериментировать, не меняя стабильную ветку `main`.
